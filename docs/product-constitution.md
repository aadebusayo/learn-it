# Product Constitution

Learn It is a cognitive operating system for deliberate learning. The system exists to transform dense information into mastery.

## Prime Directive

The system never removes productive struggle. It removes unnecessary friction.

## Optimization Targets

1. Durable understanding over completion.
2. Retrieval over recognition.
3. Application over memorization.
4. Productive struggle over convenience.
5. Long-term retention over short-term satisfaction.

## Learning Loop

Every concept follows the same lifecycle:

```text
Discover -> Diagnose -> Learn -> Apply -> Explain -> Challenge -> Retain
```

## Depth Model

```text
Level 0: Recognition
Level 1: Explanation
Level 2: Application
Level 3: Derivation
Level 4: Teaching
Level 5: Innovation
```

Users do not complete concepts. They deepen them.

## Topic Representation

Every topic should be represented as:

- Problem
- Intuition
- Mathematics
- Implementation
- Limitations
- Historical context

## Assessment Signals

Understanding is evaluated through:

- Explanation quality
- Transfer ability
- Retrieval accuracy
- Implementation success
- Edge-case handling
- Concept connections created

## Anti-Goals

- Passive summary generation
- Chat-with-PDF as the main product
- Coins, streak obsession, XP inflation, or arbitrary badges
- Answer-first tutoring
- Engagement metrics that do not reflect learning
